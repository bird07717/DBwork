# MVP v2 演示指南

本文档用于 MVP v2 答辩演示、截图和最终验收。

## 1. 启动顺序

在项目根目录启动基础设施：

```bash
docker compose up -d
docker compose ps
```

分别在三个终端启动服务：

```bash
make backend
make ai
make frontend
```

访问入口：

- 前端：`http://localhost:5173`
- 后端健康检查：`http://localhost:8080/api/health`
- AI 健康检查：`http://localhost:8000/health`

## 2. 推荐演示链路

1. 打开首页态势看板，展示全局线路数、区间客流、最高满载率、热点站点和地图态势。
2. 切换 1/2/3 路，说明左侧浮层和右侧站点压力使用当前线路口径。
3. 点击热点站点，展示站点压力颜色和站点详情。
4. 进入客流分析页，展示日客流趋势、高峰/平峰对比、线路对比和站点排行。
5. 阅读“自动分析说明”，说明系统不只是画图，还能解释图表。
6. 进入 AI 助手页，选择线路和日期，生成结构化调度分析报告。
7. 登录管理员后台，进入调度建议管理。
8. 选择线路、日期和时段生成建议，展示满载率、客流量、热点站点、规则依据和 AI 风险提示。
9. 展示历史建议记录展开详情，说明建议可追溯。

## 3. 截图清单

建议准备以下截图：

1. 首页态势看板全图。
2. 首页线路切换和站点压力图。
3. 首页告警摘要区域。
4. 客流分析页日客流趋势图。
5. 客流分析页高峰/平峰对比图。
6. 客流分析页线路对比图。
7. 客流分析页自动分析说明。
8. AI 助手结构化调度分析报告。
9. 后台调度建议生成结果。
10. 后台调度建议热点站点和规则依据。
11. 后台调度建议历史详情。
12. 数据库 ER 图或表结构截图。
13. Swagger / Knife4j 接口文档截图。

## 4. 讲解口径

线路和站点来自真实线路数据；运营客流为基于调查口径和业务规则生成的模拟数据，用于课程项目演示和统计分析验证。

AI 报告基于后端传入的结构化统计数据生成；未配置模型 Key 或模型调用失败时，系统会自动回退到本地规则分析，保证演示稳定。

当前系统未接入实时车辆 GPS，也未接入真实刷卡支付系统，因此答辩时不要表述为实时车辆定位或真实刷卡数据分析。
