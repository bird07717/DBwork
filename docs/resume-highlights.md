# 简历与答辩亮点

## 项目一句话

城市公交客流分析与智能调度 Agent 平台：基于 Vue 3、Spring Boot、FastAPI、MySQL 和 Redis 构建的公交客流分析与调度建议系统，支持地图态势、图表分析、AI 报告和管理员调度建议闭环。

## 简历写法

```text
城市公交客流分析与智能调度平台
- 基于 Vue3 + Spring Boot + FastAPI + MySQL 构建前后端分离公交客流分析系统，支持游客地图态势展示、管理员数据维护、客流统计和 AI 调度建议。
- 设计公交线路、站点、班次、乘车记录、调度建议等核心表结构，使用外键、唯一约束和索引支撑线路客流、站点排行、高峰满载率等统计查询。
- 实现地图驱动首页，将线路状态、站点压力、区间客流、最高满载率和告警摘要可视化展示，并支持腾讯地图 Key 缺失时降级展示。
- 实现客流分析看板，支持日客流趋势、高峰/平峰对比、线路对比、站点 TOP 排行和自动分析结论。
- 接入 Python AI 微服务和 DeepSeek 模型，基于后端统计数据生成结构化调度分析报告；模型不可用时自动回退本地规则分析。
- 实现管理员调度建议生成与历史记录展示，结合客流量、满载率、热点站点和规则阈值给出可解释调度建议。
```

## 答辩重点

- 数据库设计：线路、站点、线路站点、车辆、司机、班次、乘车记录、调度建议、管理员、操作日志构成完整业务闭环。
- 统计查询：线路日客流、站点上车排行、高峰/平峰班均客流、班次满载率支撑前端图表和调度建议。
- 前端展示：首页采用地图主视角，客流分析页用图表和自动结论解释业务问题。
- AI 能力：AI 服务不直接查数据库，而是基于 Java 后端整理后的结构化统计数据生成报告。
- 降级策略：地图 Key 缺失、AI Key 缺失或 AI 服务不可用时，系统仍可展示降级视图和规则分析。

## 风险表述

- 运营客流是模拟数据，不是官方真实刷卡数据。
- 当前版本不包含实时车辆 GPS 定位。
- AI 报告是辅助解释，最终调度仍需结合车辆、司机、道路和实际运营情况。
